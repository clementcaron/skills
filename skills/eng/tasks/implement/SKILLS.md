Implement the work described by the user in the PRD, slice document, or approved slice section.

Use /tdd where possible, at pre-agreed seams.

Run typechecking regularly, single test files regularly, and the full test suite once at the end.

Once done, review the work carefully before handing it back.

Do not create commits automatically. When the implementation is done, summarise the change and leave any commit or history-management step to the user unless they explicitly ask for it.

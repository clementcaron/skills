# Skills that I use

This repository contains a collection of AI agent skills optimized for a workflow where **the human remains the strategic "General"** and the AI acts as a tactical **"Sergeant"**.

These skills are rewritten to **never interact directly with GitHub or perform automated commits**. The goal is to keep repository maintenance manual while still preserving durable architectural context.

## Repository architecture

The project is organized by language and development phase:

- **`skills/eng/planning/`**: For extracting ideas, aligning on domain models, setting up repo policy, and running research spikes via prototypes.
- **`skills/eng/tasks/`**: For turning approved plans into PRDs, slices, implementations, and session handoffs.
- **`skills/eng/maintenance/`**: For fighting software entropy, deepening shallow modules, and handling maintenance workflows without automating history operations.

## Development routine

2. **Tasks**
3. **Maintenance**
